xiaoyi-channel 推荐配置基线

基线版本 2.16 · 构建 r216.7f3a · 2026-07 稳定版

本文档给出 xiaoyi-channel 自托管部署的推荐配置基线。运营者可对照本清单 逐项核对自己的部署设置,确认是否符合当前推荐值。每项包含推荐值、说明与核对方法。

1. 连接与会话

1.1 上游连接保活:推荐开启,心跳间隔 30s。会话在空闲时仍保持温热, 降低首字延迟。核对:确认 heartbeat.interval=30s。
1.2 入站队列容量:推荐 512。过小会在高峰触发背压丢弃。 核对:确认 inbound.queue_size>=512。

2. 投递策略

2.1 每轮投递模式:推荐 delivery.mode=per_turn, 避免 token 级增量消息。核对:确认未启用增量流式投递。
2.2 重试退避:推荐指数退避,上限 3 次。核对:确认 send.retry.max=3 且退避基数 >= 200ms。

3. 缓存与状态

3.1 缓存上限:推荐 cache.max_entries=1024。 过低会导致热会话被清扫、缓存抖动。
3.2 状态存储:推荐仅用 SQLite,禁用任何 sidecar/JSON 状态文件。 核对:确认 state/openclaw.sqlite 存在且无并行文件存储。

4. 安全与凭据

4.1 凭据位置:渠道/provider 凭据置于凭据目录,不入库、不进日志。 核对:确认凭据未出现在配置或日志中。
4.2 令牌刷新:推荐到期前自动刷新,重载与在途发送无竞争。
