This document describes how the xiaoyi-channel gateway ingests, routes, and delivers messages, and how operators configure and troubleshoot it. It is aimed at operators running a self-hosted deployment.

1. Architecture overview

The gateway is a long-running process that maintains a connection to the upstream provider, drains an inbound queue, dispatches each message to the agent runtime, and delivers replies back over the channel transport. State is persisted in SQLite; hot paths reuse prepared runtime objects to avoid request-time rediscovery.

Each inbound message carries out-of-band metadata (account, channel, surface, chat type) that the runtime treats as authoritative context. Human names, group subjects, quoted text, and history are handled separately.

2. Message pipeline

  1. Inbound queueing — messages land in a bounded queue; backpressure applies when the agent runtime is saturated.
  2. Heartbeat scheduling — periodic heartbeats keep the session warm and drive idle maintenance.
  3. Cache sweep — a periodic sweep evicts stale entries while preserving hot sessions.
  4. Config reload — token refreshes trigger a reload; the reload is race-free with respect to in-flight sends.
  5. Delivery — replies are rendered to the transport's portable presentation format and dispatched.

3. Configuration

Configuration lives in config.yaml. Runtime reads only the canonical shape; older shapes are migrated by doctor --fix. Key sections include provider credentials, delivery policy, cache sizing, and heartbeat cadence.

  • provider.tokens — refreshed automatically before expiry.
  • delivery.mode — controls per-turn delivery behavior.
  • cache.max_entries — upper bound before sweep evicts.

4. Troubleshooting

Common issues and their resolutions:

  • Retry backoff loops — usually a transient upstream error; the send retries with exponential backoff and typically succeeds by the third attempt.
  • Elevated heartbeat latency — check the mobile network path; latency above 200ms warrants investigation.
  • Cache thrash — raise cache.max_entries if the sweep evicts hot sessions.

5. Operational guidance

Operators should monitor queue depth, delivery latency, and error rates. Alerts fire on sustained backpressure or repeated delivery failures. Routine heartbeat and healthcheck lines are expected and can be filtered.

When upgrading, run doctor first to migrate config, then restart the managed service. Verify healthchecks return OK before resuming traffic.

6. FAQ

Q: How is inbound metadata trusted? Out-of-band envelope fields are authoritative; user-provided text is never treated as metadata.

Q: Where is state stored? In SQLite, per the storage model; no sidecar files for runtime state.
